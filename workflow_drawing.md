# Workflow: 工程圖 PDF → FCStd

這是**最高精度**的輸入來源，因為工程圖上的尺寸都是精確的。

## 步驟

### Step 1: 讀 PDF
使用 Read tool 讀 PDF（支援多頁）。注意：
- 三視圖通常在同一頁
- 剖面圖 (Section) 揭露內部特徵
- 局部放大圖 (Detail) 顯示小特徵
- 標題欄有料號、材質、公差等級、比例

### Step 2: 萃取尺寸清單
從標註萃取所有尺寸，包括：
- 線性尺寸（長度、寬度、深度）
- 直徑 ⌀xx、半徑 Rxx
- 角度
- 螺紋規格（M4 x 0.7、M6 x 1.0）
- 公差（H7, g6, ±0.05）
- GD&T 符號（垂直度 ⊥、平行度 ∥、位置度 ⌖）

輸出表格給使用者確認：
```
| 尺寸代號 | 數值 | 適用特徵 |
|---|---|---|
| ⌀80 h7  | 80 -0/-0.03 | 外徑 |
| ⌀45 H7  | 45 +0.025/0 | 內孔 |
| M4 x 4  | M4 深 8   | 端面螺紋孔 |
```

### Step 3: 視圖對應
- 正視圖 → XZ plane
- 俯視圖 → XY plane
- 左/右視圖 → YZ plane
- 剖面圖 → 內部特徵的依據

### Step 4: 建立 spec.json
把尺寸對應到特徵：
- 旋轉對稱 → Revolution base
- 多孔陣列 → 單孔 + PolarPattern
- 對稱槽 → Mirror 或 PolarPattern
- 所有螺紋 → Hole feature（不用 Pocket）

### Step 5-7: 同 photo workflow

## 特殊處理：手繪草圖

若使用者給手繪草圖：
1. 辨識度比工程圖差，尺寸標註常不完整
2. 先列出「我看得到的尺寸」+ 「我需要但沒寫的尺寸」
3. 請使用者補全關鍵尺寸才進 spec

## 特殊處理：多頁 PDF

若 PDF 多頁：
- 通常 Page 1 是總覽，Page 2+ 是子特徵放大
- 先讀全部再統整 spec
- 組立圖 (assembly drawing) 要拆成多個 spec（每個零件一個）
