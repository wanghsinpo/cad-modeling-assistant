# Workflow: 照片 → FCStd

## 輸入需求（告訴使用者怎麼給才有用）

**必要**：
- 至少 **2 張照片**（正視圖角度 + 斜 45°）。3 張最佳（正 + 側 + 斜）
- **1 個已知基準尺寸**（最好是外徑或總長）

**加分**：
- 拍照時將零件放在方格紙上（提供比例參考）
- 同一張照片含游標卡尺正在量某段（可直接讀尺寸）
- 背景乾淨、光源均勻、零件佔畫面 60% 以上

**常見誤差來源（要跟使用者講）**：
- 單張照片 → 深度維度全靠猜
- 透視變形 → 距離測量會有 ±5% 誤差
- 被遮擋面（背面、內部）→ 需要使用者補拍或描述

## 執行步驟

### Step 1: 視覺判讀
從照片提取：
- 零件整體形狀類別（軸/環/板/殼體…）
- 可見特徵：孔、槽、凸台、倒角、螺紋、花鍵
- 特徵數量（目測）與大致分布（對稱？環狀？）
- 可見的標記（螺紋規格 M?, 料號, 材質）

### Step 2: 比例校正
- 使用者給的已知尺寸 → 計算照片 pixel/mm 比例
- 反推其他可見特徵的尺寸
- **誤差容許度**：告訴使用者這是估計值，±10% 是正常的

### Step 3: 結構化特徵清單（強制對齊）
輸出類似：
```
零件類型: 環狀定位環
外徑: 80 (來自你給的已知尺寸)
總高: 估計 25 (從照片比例推算, ±3)
內孔: 估計 45
可見特徵:
  A. 上端面 4 個 M4 螺紋孔，均分（bolt circle R32）
  B. 外緣 4 個徑向矩形凹槽（cast. notches），與 A 錯開 45°
  C. 外圓柱面 4 個徑向小孔（推測銷孔 ⌀3）
不確定:
  - 下端面特徵（被遮擋）
  - 螺紋深度
  - 是否有倒角
```

等使用者確認 / 補充後再進 Step 4。

### Step 4: 生成 spec.json
依照 [spec_schema.md](spec_schema.md) 格式。

### Step 5: 呼叫 generator
```bash
python3 ~/.claude/skills/cad-modeling-assistant/tools/generate_macro.py spec.json out.FCMacro
```

### Step 6: 指示使用者執行
1. 把 `out.FCMacro` 複製到 FreeCAD macro 目錄
2. FreeCAD → Macro → Macros → 選擇 → Execute
3. 另存為 FCStd

### Step 7: 收集回饋並迭代
```
使用者回報錯誤 → 更新 lessons_learned.md → 調整 spec.json → 重新生成
```

## 單張照片的特殊處理

只有一張照片時：
- 只信賴**主視角**能看到的特徵
- 對稱性用來推論看不見的一面（「如果上下對稱，我假設下面 = 上面」）
- 明確告訴使用者「我假設 X、Y、Z，可能錯，請確認」
- 不鼓勵只給一張照片 → 建議補拍

## 從影片抽取

若使用者給影片：
1. 用 `ffmpeg -i video.mp4 -vf "fps=1" frame_%03d.png` 抽出 1 FPS 的 frame
2. 人工挑選 3–5 個最佳角度的 frame
3. 按照多張照片流程處理
